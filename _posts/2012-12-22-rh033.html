七，深入介绍文件系统

找文件过程

directory -> FCB(inode number) -> 内存的inode找 -> 没有就通过inode分区创建 -> 逻辑地址 -> 物理地址

 

block 最小单位 = 4k

disk -> partition -> filesystems ext3

也支持etx2 msdoc iso9660 (cd)

 

inode = index node

 

决定最多block

time stamp

:access time

:modify time

:change time

 

directory中FCB分为：1. file name 2. inode-no

ls -li 看 directories 中FCB信息

cp and inodes 创建新的inode

找个空的innode-no -> 写入inode talbe -> directories (inode-no 与 file name对应) -> cp到block中

mv and inodes

mv同一个分区，不变inode

不同一个分区,变inode

rm and inodes

rm后 link count = 2 变为 1

把diretory中的内容删除 不是真的删除

rm 文件删除时inode还是存在的，可以恢复

只是删除了FCB, inode还是在的所以文件是在的

 

symbolic link ：inode-number不同, 文件大小一样 (相当一个新的文件)

ln -s filename [linkname]

lrwxrwxrwx

 

hard link ：inode-number相同 (同一个文件，一个inode文件，多个FCB指向同一个inode)

hard link 多文件指向同一个inode 就是多个文件的inode号一样

只能对文件 不能对目录做

不同file name 相同的inode-no

避免东西被删
